Problem 2: Caesar Cipher (3 points)

The caesar cipher is a (very outdated...) method of encryption that uses a key, also known as a shift, to encrypt messages.
To encrypt any letter of the alphabet, you simply add the shift to it. For example, with a shift of 3, a becomes a + 3 which is d, b becomes e, etc. Some more info on the notes from our cryptography workshop
Brutus wants to figure out what key Caesar most commonly uses. He somehow has access to several messages and their encrypted counterparts. Write a program to help Brutus figure out what key was used to encrypt a certain message.

Specifications

Input1: a plaintext String
Input2: another string that is the encrypted form of Input1
Output: The key used to encrypt Input1 into Input2
Assume that only letters a-z will be in the input. No need to worry about punctuation or capital letters.
The encryption scheme wraps around, i.e. z + 3 = c, c + 25 = b, z + 20 = t
You can assume that Input2 will always be the correct encrypted form of Input1.
Note: The ascii range for a-z starts with a at 97. If you want to work with a-z as 1-26 subtract 96 from the char.

Example Run

Plain text: gopokes
Encrypted text: ltutpjx
Key Used: 5

Problem 4: So many integer sequences... (3 points)

The fibonacci sequence F(n) is defined as follows:
F(n) = F(n-1) + F(n-2) with F(0) = 0 and F(1) = 1 (more info)
Recamán's sequence is defined as follows:
a(0) = 0; for n > 0, a(n) = a(n-1) - n if a(n-1)-n is positive and not already in the sequence, otherwise a(n) = a(n-1) + n. (more info)
Write a program that, when given a number x, determines if x is in the the first 45 elements of the fibonacci sequence. If x is in the first 45 elements of the fibonacci sequence, let n be equal to the index in the fibonacci sequence of the first occurence of x. i.e. with x = 2 , 2 is the 3rd element of the fibonacci sequence, so n = 3. Now that you have n determine what Recamán(n) is. If x is not in the fibonacci sequence, inform the user and quit the program.

Specifications

n <= 701408733

You only have to check if n is one of the first 45 elements of the fibonacci sequence, that is fib(0) through fib(44).

Example Runs

Input: 4181
Output: 62
Input: 701408733
Output: 36
Input: 9
Output: 9 is not in the first 45 elements of the fibonacci sequence

Problem 5: 2D Palindromes

A 2 dimensional array is a rectangular array of elements. The reverse image of a 2 dimensional array is defined to be the array reversed in both directions. For example, the reverse image of
1 4 7
2 5 8
3 6 9
is:
9 6 3
8 5 2
7 4 1
A 2 dimensional array is said to be palindromic if there is no difference between it and it's reverse image. Write a program that reads in a 2 dimensional array from a file, and checks to see if it is palindromic

Specifications

Input: File name of a file containing a 2 dimensional array, with elements seperated by spaces, rows seperated by lines.
Output: A message stating whether or not that 2 dimensional array is palindromic.

Example Run

File Name to Read: 2darray.txt
(where 2darray.txt looks like this):
1 2 3
2 1 2
3 2 1
Output: This 2 dimensional array is palindromic.
File Name to Read: 2darray.txt
(where 2darray.txt looks like this):
1 3 4
3 6 8
3 4 2
Output: This 2 dimensional array is not palindromic.

Problem 6: Collatz

The Collatz sequence is defined as:
Start with any positive integer n.
if (3n + 1) is odd:
  The next term in the sequence will be (3n + 1)
if (3n + 1) is even:
  The next term in the sequence will be n/2
We do this until we reach one.
For example, the Collatz sequence of 3 is:
3 -> 10 -> 5 -> 16 -> 8 -> 4 -> 2 -> 1
The length of the above Collatz sequence is 8
Write a program that, when given a length, finds the smallest positive integer whose Collatz sequence is of that length

Specifications

Input: an integer n, representing a length.
Output: The smallest positive integer whose Collatz sequence is of that length.

Example Run

Length: 3
Smallest Positive Integer with Length 3: 4
(because the collatz sequence of 4 is 4 -> 2 -> 1, which is of length 3)
