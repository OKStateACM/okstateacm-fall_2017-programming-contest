Problem 2: Caesar Cipher (3 points)

Instructions

The caesar cipher is a (very outdated...) method of encryption that uses a key, also known as a shift, to encrypt messages.
To encrypt any letter of the alphabet, you simply add the shift to it. For example, with a shift of 3, a becomes a + 3 which is d, b becomes e, etc. Some more info on the notes from our cryptography workshop
Brutus wants to figure out what key Caesar most commonly uses. He somehow has access to several messages and their encrypted counterparts. Write a program to help Brutus figure out what key was used to encrypt a certain message.

Specifications

Input1: a plaintext String
Input2: another string that is the encrypted form of Input1
Output: The key used to encrypt Input1 into Input2
Assume that only letters a-z will be in the input. No need to worry about punctuation or capital letters.
The encryption scheme wraps around, i.e. z + 3 = c, c + 25 = b, z + 20 = t
You can assume that Input2 will always be the correct encrypted form of Input1.
Note: The ascii range for a-z starts with a at 97. If you want to work with a-z as 1-26 subtract 96 from the char.

Example Run

Plain text: gopokes
Encrypted text: ltutpjx
Key Used: 5

Problem 4: 2D Palindromes (6 points)

Instructions

A 2 dimensional array is a rectangular array of elements. The reverse image of a 2 dimensional array is defined to be the array reversed in both directions. For example, the reverse image of
1 4 7
2 5 8
3 6 9
is:
9 6 3
8 5 2
7 4 1
A 2 dimensional array is said to be palindromic if there is no difference between it and it's reverse image. Write a program that reads in a 2 dimensional array from a file whose filename is specified by either stdin or a command line argument, and checks to see if that 2 dimensional array is palindromic

Specifications

Input: File name of a file containing a 2 dimensional array (either parsed as a command line argument or read from stdin)
File: The file should contain a 2 dimensional array of elements, where each row is a line, and each element in each row is seperated by a space.
Output: A message stating whether or not that 2 dimensional array is palindromic.

Example Run

File Name to Read: 2darray.txt
(where 2darray.txt looks like this):
1 2 3
2 1 2
3 2 1
Output: This 2 dimensional array is palindromic.
File Name to Read: 2darray.txt
(where 2darray.txt looks like this):
1 3 4
3 6 8
3 4 2
Output: This 2 dimensional array is not palindromic.

Problem 6: Collatz (7 points)

Instructions

The Collatz sequence is defined as:
Start with any positive integer n.
if (3n + 1) is odd:
  The next term in the sequence will be (3n + 1)
if (3n + 1) is even:
  The next term in the sequence will be n/2
We do this until we reach one.
For example, the Collatz sequence of 3 is:
3 -> 10 -> 5 -> 16 -> 8 -> 4 -> 2 -> 1
The length of the above Collatz sequence is 8
Write a program that, when given a length, finds the smallest positive integer whose Collatz sequence is of that length

Specifications

Input: an integer n, representing a length.
Output: The smallest positive integer whose Collatz sequence is of that length.

Example Run

Length: 3
Smallest Positive Integer with Length 3: 4
(because the collatz sequence of 4 is 4 -> 2 -> 1, which is of length 3)

Problem 9: On The Grid (What the Funge?)

Befunge is an esoteric programming language that uses a grid system. It was designed specifically to be the hardest programming language to write a compiler for. Each instruction is part of a grid. The interpreter starts at the top left (0, 0), runs the instruction, then moves right(unless told otherwise) and runs the instruction at that cell. A Fungeoid is any language that is in the befunge family. It must use a grid system, like befunge does. We have defined a simple fungeoid of which the permitted operations are listed below.
R Start moving right
L Start moving left
U Start moving up
D Start moving down
0-9 Push this number onto the stack
" Start string mode: push each character's ASCII value all the way up until the next "
P Pop value from the stack and output as ASCII character
_ no op: proceed to next cell without doing anything
J One time jump: Pop a value off the stack which becomes x. Pop another value off the stack which becomes y. Move to cell (x,y) and proceed execution. If you return to the cell in which the J instruction was originally executed, do not execute it again.
Q Stop Execution
Each one of these operations would exist inside a cell on the grid system. An example code snippet would be:
R"a"_PD
Q_J00_L
It's a good idea to visualize this as a grid:
R " a " _ P D
Q _ J 0 0 _ L
  • Execution starts at (0, 0) with R. This tells the interpreter to start moving right. (This isn't actually necessary right now, since execution should move to the right by default, but it's important for later.)
  • Moving to the right, the next cell contains " which tells the interpreter to enter string mode. Execution continues to move to the right.
  • The next cell contains an 'a'. Since it is in string mode, the 'a' is pushed onto the stack.
  • The next cell contains ", which terminates string mode.
  • The next cell is a no op, nothing happens and the interpreter moves right one
  • The next cell contains an operation, P, which tells the interpreter to pop the top value off of the stack and print it to stdout.
  • Still moving to the right, the next cell contains an operation D, which tells the interpreter to start moving down the grid cell by cell.
  • The interpreter moves down one cell. The operation in this cell is L, which tells the interpreter to start moving left cell by cell.
  • The interpreter moves to the left one cell. The _ in this cell represents a "no op", meaning nothing will happen, and the interpreter will continue moving in the direction it was previously.
  • The interpreter moves left one cell. The 0 in this cell tells the interpreter to push a 0 onto the stack.
  • The interpreter moves left one cell and again pushs 0 to the stack.
  • The interpreter once again moves left one cell. The J operation tells the interpreter to pop a value from the stack, call it x, pop the next value from the stack, call it y, and move to (x, y). In this case, since the last two values pushed to the stack were 0 and 0, the interpreter will move to (0, 0).
  • It will go back to (0, 0). The direction the interpreter was moving before the J operation was left, so after moving to (0, 0) and executing that command it should move left one cell. However, the operation in cell (0, 0) is R, which tells the interpreter to start moving right cell by cell
  • Execution will therefore move to the right, and everything will be the same as before. until...
  • When the interpreter reaches the J instruction, it should not execute it, and execution should continue moving.
  • Moving left again, the interpreter encounters a _(no op), and then a Q, which stops execution
Write an interpreter for this language. It should read in a file and execute the code specified in the file.

Specifications

Input: A file (specified as a command line argument) which contains valid code.
Output: Whatever the output of the code specified in the file is.

Example Runs

Input File:
"dlroW olleH"PPPPPPPPPPPQ
Output:
Hello World
Input File:
_________D
D"Hello"_L
RPPPPP18JQ
Output:
